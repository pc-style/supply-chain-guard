Supply Chain Guard scans npm packages and VS Code extensions before they touch your machine. Lifecycle scripts, credential reads, suspicious payloads, Socket signals — all locally.

  • Malicious scripts and hidden actions
  • bun, npm, yarn, pnpm, VS Code
  • Fast, local, no telemetry
VERY VERY EARLY STAGE. Can miss malware or flag safe packages. Warning layer only — not proof of safety.

// WHAT_IT_CHECKS

Lifecycle scripts

preinstall, install, postinstall, prepare

Remote downloads

curl, wget, fetch

File system access

reads, writes, sensitive paths

Network activity

external host connections

Policy & licenses

CVEs, allow / block rules

Socket signal

optional socket.dev score

Also: VS Code extension internals and optional Codex/PI review ending with SCGUARD_DECISION: approve.

// HOW_IT_WORKS

Small loop. All on disk.

Install does not start until scan finishes and approval is explicit. Nothing leaves your machine unless you request Socket lookup.

  1. 01

    Stage

    Resolve tarball or .vsix, cache to .scguard/cache.

  2. 02

    Scan

    Heuristics over staged files, optional Socket, risk score.

  3. 03

    Decide

    JSON/Markdown report or agent review. Approve to continue.

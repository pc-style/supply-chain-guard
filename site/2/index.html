A local review before every install

Review before you install.

Supply Chain Guard scans npm packages and VS Code extensions before they touch your machine — lifecycle scripts, credential reads, suspicious payloads, and Socket signals, all locally.

  • Malicious scripts and hidden actions
  • bun, npm, yarn, pnpm, and VS Code
  • Fast, local, no telemetry
Very early stage. May miss malware or flag safe packages. A warning layer, not proof of safety.

What it checks

Six lenses on every package.

Lifecycle scripts

preinstall, install, postinstall, prepare

Remote downloads

curl, wget, fetch

File system access

reads, writes, sensitive paths

Network activity

connections to external hosts

Policy & licenses

CVEs, allow / block rules

Socket signal

optional socket.dev score

Also VS Code extension internals and optional agent review with SCGUARD_DECISION: approve.

How it works

A small loop, all on disk.

Install waits for scan and explicit approval. Nothing leaves your machine unless you request Socket.

  1. Stage

    Tarball or .vsix to .scguard/cache.

  2. Scan

    Heuristics, optional Socket, risk score.

  3. Decide

    Report or agent review — approve to continue.
